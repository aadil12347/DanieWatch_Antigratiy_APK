## Plan: Enhance Top Navbar with Animated Active Indicator

**Goal**: Improve the `TopNavbar` widget on the Explore page so that the active option:
- Appears larger and slightly pushes surrounding options away.
- Shows an animated glowing underline that moves smoothly when the user switches options.
- Uses the app's default red colour for the glow (matching the existing theme).
- Provides a buttery‑smooth animation without lag.

---

### 📚 Research (read‑only)
1. Find Flutter examples for:
   - Custom `TabBar` indicator with gradient/glow.
   - Animated underline with `AnimatedPositioned` / `AnimatedContainer`.
   - Performance‑friendly scaling of selected tab text.
2. Verify the default red colour defined in the app's theme (search `ThemeData` or colour constants).
   - Use `grep` to locate the colour definition (e.g., `primaryColor`, `appRed`).
   - Confirm the exact hex/value for consistency.

---

### 🛠️ Implementation Steps
#### 0️⃣ Preparation (Read)
- 📄 Read `lib/presentation/widgets/top_navbar.dart` (already reviewed).
- 📄 Read the theme file (e.g., `lib/presentation/theme.dart` or wherever colours are defined) to capture the red colour value.

#### 1️⃣ Choose Implementation Approach
- **Option A – Replace ListView with `TabBar`**:
  - Use `DefaultTabController` with `isScrollable: true`.
  - Supply `Tab`s based on `TopNavbar.items`.
  - Implement a custom `UnderlineGlowIndicator` (`Decoration` subclass) that draws a red glowing line.
  - Use `labelStyle`/`unselectedLabelStyle` for size differences (e.g., 17 sp vs 15 sp) and `AnimatedDefaultTextStyle` for smooth scaling.
- **Option B – Keep ListView and Add Animated Underline**:
  - Keep existing `ListView.builder`.
  - Wrap each item in a `Column` containing the text and a container for the underline.
  - Use `AnimatedContainer` (or `AnimatedPositioned` with a single shared underline) to animate the underline's `left` offset and width.
  - Apply a `BoxShadow` with the red colour to create the glow effect.
- **Decision**: Choose **Option A** because `TabBar` already handles tab selection animation, reduces manual layout calculations, and yields better performance.

#### 2️⃣ Add Custom Indicator
- Create a new file `lib/presentation/widgets/underline_glow_indicator.dart`:
  - Define `class UnderlineGlowIndicator extends Decoration`.
  - Implement `BoxPainter` to draw a line with the app's red colour and a `BoxShadow` for the glow.
  - Use a linear gradient or `Paint` with `MaskFilter.blur` for the glow.
  - Ensure the indicator animates with `TabBarIndicatorSize.label`.

#### 3️⃣ Update `TopNavbar` Widget
- Replace the `ListView.builder` with a `TabBar` inside a `DefaultTabController`.
- Set `controller.length = items.length` and `initialIndex = selectedIndex` (computed from filters, same logic as before).
- Provide `tabs: items.map((label) => Tab(text: label)).toList()`.
- Configure `TabBar`:
  ```dart
  TabBar(
    isScrollable: true,
    labelStyle: GoogleFonts.inter(fontSize: 17, fontWeight: FontWeight.bold),
    unselectedLabelStyle: GoogleFonts.inter(fontSize: 15, fontWeight: FontWeight.w500),
    labelColor: Colors.white,
    unselectedLabelColor: Colors.white.withOpacity(0.5),
    indicator: UnderlineGlowIndicator(),
    indicatorSize: TabBarIndicatorSize.label,
    onTap: (index) => _onItemTapped(ref, items[index], filters),
    // Add animation duration if needed (default is 200ms, can customise via TabBarTheme)
  )
  ```
- Remove the `AnimatedScale` and `AnimatedDefaultTextStyle` wrappers (TabBar will handle sizing). If additional scaling is desired, wrap the `Tab`'s child in an `AnimatedScale` that uses `TabController.animation` to drive the scale.

#### 4️⃣ Adjust Tap Handling
- Keep `_onItemTapped` unchanged; now `TabBar.onTap` will call it with the selected label.
- Ensure that tapping "Explore" clears filters as before.

#### 5️⃣ Integrate Theme Red Colour
- Import the colour constant (e.g., `import '../theme.dart';`).
- Use that colour in `UnderlineGlowIndicator` and any shadows.

#### 6️⃣ Performance Optimisations
- Use `const` for static widgets where possible.
- Ensure `TopNavbar` remains a `ConsumerWidget` to only rebuild when `searchProvider` changes.
- Verify that the TabBar's internal animation works at 60 fps (test on device).
- If lag remains, consider wrapping the TabBar in `RepaintBoundary`.

#### 7️⃣ Testing
- **Unit/Widget Tests**:
  - Verify that the correct tab is selected based on `searchProvider` state.
  - Simulate taps and assert that `searchProvider` updates accordingly.
  - Check that the underline moves to the selected tab.
- **Manual UI Test**:
  - Run the app on an emulator/device.
  - Switch between each option, observe the scaling, underline animation, and any lag.
  - Confirm the underline colour matches the app red.

#### 8️⃣ Documentation & Clean‑up
- Add a brief comment at the top of `underline_glow_indicator.dart` describing its purpose.
- Update any existing documentation (e.g., `README` or widget docs) to mention the new navigation behavior.
- Run `flutter analyze` and `flutter test` to ensure no lint or test failures.
